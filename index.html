JS 進階概念

1. 函式的三種寫法

// 傳統寫法
function add(n1, n2){ return n1 + n2; }
// 變數指定
let add2 = function(n1, n2){ return n1 + n2; };
// 箭頭函式
const add3 = (n1, n2) => { return n1 + n2; };

2. JSON 與參數物件傳遞

function multiply(args){ return args.n1 * args.n2; }
let ans = multiply({ n1: 3, n2: 5 });
console.log(ans); // 15

3. 物件打包多個回傳值

function addAndMultiply(args){
    return { addAns: args.n1 + args.n2, mulAns: args.n1 * args.n2 };
}
let ans = addAndMultiply({ n1: 3, n2: 5 });
console.log(ans.mulAns);

JS 模組化開發

在 Node.js 中，透過 import 和 export，我們可以將程式拆解到不同的檔案中，讓程式碼更乾淨、更好維護。

1. 檔案分工範例

[ lib.js ] (工具箱檔案) 
export const add = (n1, n2) => n1 + n2;
export const mul = (n1, n2) => n1 * n2;
export default "這是一個模組";
[ main.js ] (主程式檔案) 
import { add, mul } from "./lib.js";
console.log(add(3, 4)); // 7

2. Export 的三種實戰用法

我們除了可以匯出物件，還可以自由決定要匯出什麼：

  • 具名匯出 (Named Export)：使用 export 放在變數或函式前，一次可以匯出多個，匯入時必須用 { }。
  • 預設匯出 (Default Export)：一個檔案只能有一個 export default。匯入時可以「自訂名稱」，不需要 { }。
  • 統一匯出：在檔案最後統一整理要匯出的內容，這對大型專案管理最整潔。
// 統一匯出寫法
const add = (a, b) => a + b;
const mul = (a, b) => a * b;
export { add, mul }; // 一次匯出多個
