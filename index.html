Module 01

JavaScript 核心基礎

掌握變數、流程控制、函式與物件，是所有 JS 開發的起點。

1

變數與資料型態

變數是存放資料的容器。let 可重新賦值；const 宣告後不可更改。常見型態包含數字（Number）、字串（String）與布林（Boolean）。

let score = 80;
score = 90;
console.log(score);
2

流程控制

if...else 處理條件分支；for 迴圈處理重複工作。條件為 false 時執行 else 區塊。

let i = 0;
if (i > 0) { console.log("A"); }
else       { console.log("B"); }
3

函式

函式將程式碼打包成可重複呼叫的單元，透過 ( ) 傳入參數，以 return 回傳結果。

const multiply = (a) => { return a * 2; };
console.log(multiply(5));
4

物件

物件用 { } 包裹，由「鍵（Key）」與「值（Value）」組成，透過 . 存取屬性。

let car = { color: "Red" };
console.log(car.color);
Module 02

JS 進階概念

了解函式的多種寫法，以及如何利用物件傳遞與回傳複雜資料。

1

函式的三種寫法

JavaScript 提供三種定義函式的方式，結果相同但語法不同。選擇哪種風格因情境而異。

// 傳統宣告式
function add(n1, n2) { return n1 + n2; }

// 函式表達式
let add2 = function(n1, n2) { return n1 + n2; };

// 箭頭函式（現代首選）
const add3 = (n1, n2) => { return n1 + n2; };
2

JSON 與參數物件傳遞

將多個參數包進物件再傳入函式，讓呼叫端更清晰易讀，也方便未來新增欄位。

function multiply(args) { return args.n1 * args.n2; }

let ans = multiply({ n1: 3, n2: 5 });
console.log(ans); // 15
3

物件打包多個回傳值

JavaScript 函式只能 return 一個值，但可以把多個結果打包成物件一次回傳。

function addAndMultiply(args) {
    return {
        addAns: args.n1 + args.n2,
        mulAns: args.n1 * args.n2
    };
}
let ans = addAndMultiply({ n1: 3, n2: 5 });
console.log(ans.mulAns);
Module 03

JS 模組化開發

透過 import 與 export，將程式拆解至不同檔案，讓專案結構更乾淨。

1

檔案分工範例

一個檔案負責定義工具函式並匯出，另一個主程式檔案匯入後使用。

lib.js — 工具函式 
export const add = (n1, n2) => n1 + n2;
export const mul = (n1, n2) => n1 * n2;
export default "這是一個模組";
main.js — 主程式 
import { add, mul } from "./lib.js";
console.log(add(3, 4)); // 7
2

Export 的三種用法

  • 具名匯出（Named Export）：export 放在變數或函式前，可匯出多個，匯入時需加 { }。
  • 預設匯出（Default Export）：一個檔案只能有一個 export default，匯入時可自訂名稱，不需 { }。
  • 統一匯出：在檔案底部統一整理所有要匯出的項目，大型專案首選。
// 統一匯出寫法（推薦用於大型專案）
const add = (a, b) => a + b;
const mul = (a, b) => a * b;

export { add, mul }; // 一次匯出多個

Module 04 · 實作練習

node:fs 檔案系統模組

學習如何用 Node.js 內建的 node:fs 模組讀寫檔案。這是後端開發最常用的能力之一。

1

認識 node:fs — 什麼是檔案系統模組？

node:fs 是 Node.js 的內建模組，讓你可以在程式中操作電腦上的檔案，包含：寫入、讀取、刪除、重新命名等。

使用前需要先用 import 匯入，注意模組名稱是 "node:fs"（帶有 node: 前綴，這是現代 Node.js 的推薦寫法）：

import { writeFileSync, readFileSync } from "node:fs";
2

writeFileSync — 寫入檔案

writeFileSync(檔名, 內容) 會將字串寫入指定檔案。如果檔案不存在會自動建立；如果已存在，則會覆蓋原本內容。

import { writeFileSync } from "node:fs";

// 建立 data.txt 並寫入內容
writeFileSync("data.txt", "測試文件");

// 也可以加上編碼選項（預設就是 utf8）
writeFileSync("data.txt", "Hello, Node.js!", { encoding: "utf8" });
💡
Sync 是什麼意思？
函式名稱結尾的 Sync 代表「同步（Synchronous）」。程式會等待檔案寫完才繼續往下執行。相對的，writeFile()（非同步版）不會等待，需要用 callback 或 Promise 處理。
3

readFileSync — 讀取檔案

readFileSync(檔名, 編碼) 讀取並回傳檔案內容。一定要指定 "utf8" 編碼，否則會回傳原始的 Buffer（二進位格式），不是你想要的文字。

import { readFileSync } from "node:fs";

// 讀取 data.txt，回傳字串
const content = readFileSync("data.txt", "utf8");

console.log(content); // 印出檔案內容
4

完整範例 — 寫入後再讀取

把前面學到的合在一起：先寫入一個檔案，再讀回來印出。這就是你範例程式碼的完整流程：

import { writeFileSync, readFileSync } from "node:fs";

// ① 寫入檔案（建立 data.txt）
writeFileSync("data.txt", "測試文件");

// ② 讀取剛才寫的檔案
const content = readFileSync("data.txt", "utf8");

// ③ 印出內容
console.log(content); // → 測試文件
5

錯誤處理 — try / catch

Sync 系列函式在發生錯誤時（例如讀取不存在的檔案）會直接拋出例外（throw an Error），所以需要用 try...catch 包住，避免程式崩潰。

import { readFileSync } from "node:fs";

try {
    // 嘗試讀取一個不存在的檔案
    const content = readFileSync("notfound.txt", "utf8");
    console.log(content);
} catch (err) {
    // 捕捉錯誤，印出錯誤訊息
    console.log("讀取失敗：", err.message);
    // → 讀取失敗：ENOENT: no such file or directory, open 'notfound.txt'
}
⚠️
ENOENT 是什麼？
這是 Node.js 的錯誤代碼，代表 Error NO ENTry（找不到檔案或目錄）。這是讀取不存在的檔案時最常見的錯誤。
6
